PostgreSQL: Features you don't expect in a database

The story behind the project, what I learned from the Tour of Go, and why interactive learning beats documentation.

A few months ago I was learning Go. Not because I had to, but because I was curious. The language has a reputation for being simple, opinionated, and fast — and I wanted to see if that was true.

The first thing I did was open the Tour of Go. Within an hour I had written my first concurrent program. I didn't read a book. I didn't watch a course. I just... did it, in the browser, with immediate feedback.

"Learning by doing is the oldest idea in education. The web finally makes it easy to apply at scale."

The problem with Python docs

Python has excellent documentation. The official docs are thorough, well-maintained, and accurate. But they're written for reference, not for learning. If you're a complete beginner trying to understand what a generator is, you'll get a technically correct explanation that assumes you already know what it means.

There are great Python courses out there too — Codecademy, Real Python, countless YouTube channels. But they're all long. You commit to a course, not a concept.

What makes the Tour of Go special

The Tour of Go works because it respects your time. Each lesson is one idea, demonstrated with one example, runnable in one click. There's no setup, no environment issues, no "install this first". You're writing real Go code in 60 seconds.

The key ingredients

After building A Tour of Python, I think there are three things that make interactive learning work: immediate feedback, minimal friction, and deliberate pacing. Remove any one of them and the magic disappears. 

# Your first Python program
print("Hello, World!")

# Variables need no declaration
name = "Python"
version = 3.12

print(f"Welcome to {name} {version}")

Building this project taught me more about Python than I expected. Explaining something well forces you to understand it deeply. Writing the lesson on generators, for example, I realized I had been using them for years without fully grasping the execution model.

What's next

The project is still actively in development. I'm adding more lessons, improving the editor, and working on the people popup feature — interactive cards for the people who shaped Python's history.

If you're learning Python, give it a try. If you find a mistake or want to contribute, the code is on GitHub.
